<!-- SNIPPET: valid-heading-with-preset — wp:heading with fontSize slug + matching utility class
     KEY RULES:
     - Set "fontSize" in the JSON comment using the theme slug (e.g. "x-large"), NOT an inline value.
     - The matching utility class (has-{slug}-font-size) MUST appear on the <h*> element in HTML.
     - Never use inline font-size (e.g. style="font-size:1.5rem") — causes JSON/HTML mismatch.
     - Margin reset (top:"0", bottom:"0") prevents default browser heading margins from
       overriding blockGap spacing set on the parent container.

     Common slug → class pairs (from store.json scale):
       "x-small"  → has-x-small-font-size
       "small"    → has-small-font-size
       "medium"   → has-medium-font-size
       "large"    → has-large-font-size    (h3)
       "x-large"  → has-x-large-font-size  (h2)
       "xx-large" → has-xx-large-font-size (h1 / display)
-->
